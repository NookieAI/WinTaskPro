# WinTaskPro — Auto-Update Setup

WinTaskPro uses the [Tauri Updater plugin](https://v2.tauri.app/plugin/updater/) to deliver
silent, in-app updates via GitHub Releases.

---

## 1. Generate a signing key

```powershell
npx @tauri-apps/cli signer generate -w $env:USERPROFILE\.tauri\wintaskpro.key
```

This outputs two values:
- **Private key** — stored in `~/.tauri/wintaskpro.key` (keep secret, never commit)
- **Public key** — copy this into `tauri.conf.json`

---

## 2. Set the signing environment variable

Before building, export the private key:

```powershell
$env:TAURI_SIGNING_PRIVATE_KEY = Get-Content "$env:USERPROFILE\.tauri\wintaskpro.key" -Raw
```

For CI (GitHub Actions), add `TAURI_SIGNING_PRIVATE_KEY` as a repository secret.

---

## 3. Add the public key to `tauri.conf.json`

In `src-tauri/tauri.conf.json`, set the `plugins.updater.pubkey` field to the public key
string output by the generator:

```json
"plugins": {
  "updater": {
    "pubkey": "<paste public key here>",
    "endpoints": [
      "https://github.com/NookieAI/WinTaskPro/releases/latest/download/latest.json"
    ],
    "dialog": false,
    "windows": {
      "installerArgs": [],
      "installMode": "passive"
    }
  }
}
```

---

## 4. Build the application

```powershell
npm run build
```

Tauri automatically generates `.sig` signature files alongside the installer.

---

## 5. Create a GitHub Release

1. Create a new release tag (e.g. `v1.1.0`) in the GitHub UI or via CLI:
   ```powershell
   git tag v1.1.0
   git push origin v1.1.0
   ```

2. Upload the following files to the release:
   - `src-tauri/target/release/bundle/nsis/WinTaskPro_*_x64-setup.exe`
   - `src-tauri/target/release/bundle/nsis/WinTaskPro_*_x64-setup.exe.sig`
   - `latest.json` (generated by Tauri, found in `src-tauri/target/release/bundle/`)

3. The `latest.json` must be available at:
   ```
   https://github.com/NookieAI/WinTaskPro/releases/latest/download/latest.json
   ```

---

## How it works

- On startup (after a 3-second delay), WinTaskPro silently calls the `check_for_update` command.
- If a newer version is available at the endpoint, a non-intrusive banner appears in the
  bottom-right corner of the UI.
- The user can click **Install & Restart** to download and apply the update, or dismiss the
  banner.
- The update is verified against the public key before installation — ensuring only official
  releases are applied.
